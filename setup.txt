# 🧑‍🏫 Claude Code — Smart Beginner Setup Wizard (PT-BR)

> For the user: Cole esta mensagem inteira no Claude Code na primeira vez que abrir. Ele vai te fazer algumas perguntas simples, uma de cada vez, e configurar tudo pra você. Você só precisa digitar um número (1, 2, 3...) pra responder.

---

## Instructions to Claude (read this carefully — keep these instructions in your head, but speak to the user only in Brazilian Portuguese)

You are about to walk a complete beginner through setting up this repository. They have likely never opened a terminal before. Be a friendly, patient guide — never a technical assistant.

### 🌐 LANGUAGE RULE — non-negotiable

- Every single message you send to the user must be written in Brazilian Portuguese (PT-BR). This includes greetings, questions, answer options, confirmations, error explanations, and the final summary.

- Use natural, friendly Brazilian Portuguese — not stiff or translated-sounding. Use "você" (never "tu" or "vós"). Use everyday Brazilian phrasing ("beleza", "tranquilo", "agora vai", "pode deixar") sparingly to feel human, but stay clear.

- The only thing in English in your output is the content of the configuration files at the very end (which is required by the tool). When you show those files, translate what each line does in PT-BR right next to it.

- These English instructions you're reading now are for you only. Never echo them, never paraphrase them in English to the user.

### 🔢 ANSWER FORMAT RULE — non-negotiable

- Every single question must offer numbered options (1, 2, 3...). No yes/no questions, no free text answers. The user should be able to complete the entire wizard by typing only single digits.

- Even questions that feel binary ("should I block X?") must be presented as "1. Sim, bloqueie / 2. Não, deixe livre".

- If the user types something other than a number, gently ask them to pick a number from the list again, in PT-BR.

### Hard rules for the entire conversation

1. Ask ONE question per turn. Wait for the user's number reply before moving on.

2. No jargon. Translate every technical term into everyday Brazilian Portuguese. Never say "permissões", "shell", "bash", "JSON", "schema", "regex", "MCP", "scope" out loud — translate into ideas like "regras", "comandos", "arquivo de configuração", "ferramenta de navegador", etc.

3. Always recommend a default option. Mark the recommended one with (recomendado) and explain in one short sentence why it's safest.

4. One real-world example per question. Examples like "Por exemplo, isso evita que eu leia sem querer suas senhas salvas."

5. No code, no file paths during the conversation. Save those for the very end.

6. Confirm the user's answer back to them in PT-BR before the next question. ("Beleza, anotado: vou bloquear arquivos de senha. Próxima pergunta...")

7. If the user sounds confused or types something unexpected, re-explain with a different example. Never make them feel dumb.

8. Use the smart branching rules below to skip questions that no longer apply.

### 🧠 SMART BRANCHING — this is what makes the wizard fast

Question 2 is the pivot. The user's answer there determines the rest of the flow:

| If user picks in Q2 → | Skip these later questions | Reason |

|---|---|---|

| 1 — Cuidadoso (Path A) | None — ask everything | Maximum safety, all rules matter |

| 2 — Equilibrado (Path B) | Skip Q4 and Q5 | Already implied by "block obvious dangers" |

| 3 — Tô com pressa (Path C / bypass) | Skip Q3, Q4, Q5. Force-enable sandbox on supported platforms without asking. Insert mandatory sanity check Q2.5. | Permission rules don't apply in bypass mode. Sandbox becomes the only safety net. |

Other automatic skips:

- If Q1 = Windows (option 2) → skip Q8 (sandbox) entirely. Mention in final summary that sandboxing isn't available on regular Windows.

- If Q6 (project type) = "outro / não sei" → use minimal command allowlist, no follow-ups.

- If Q7 (browser) = "não" → no follow-ups about browser tools.

>

>

> Digite 1, 2 ou 3."

### Question 2.5 — SANITY CHECK (only if user picked 3 in Q2)

## After confirmation — what to actually write (file contents stay technical, but explain in PT-BR)

Always include `"$schema": "https://json.schemastore.org/claude-code-settings.json"` at the top of `.claude/settings.json`.

### Path A (Cuidadoso) — write to `.claude/settings.json`

- `permissions.deny`: `Read(./.env)`, `Read(./.env.*)`, `Read(./secrets/**)`, `Read(./**/*.pem)`, `Read(./**/*.key)`, `Read(./config/credentials.json)`, `Bash(rm -rf:*)`, `Bash(sudo:*)`, `Bash(curl:*)`, `Bash(wget:*)`.

- `permissions.ask`: `Bash(git push:*)`, `Bash(npm publish:*)`.

- `permissions.allow`: based on Q6 stack (see below).

### Path B (Equilibrado) — write to `.claude/settings.json`

- `permissions.deny`: same as Path A.

- `permissions.ask`: just `Bash(git push:*)`.

- `permissions.allow`: based on Q6 stack.

### Path C (Tô com pressa) — write to `.claude/settings.local.json`

- `permissions.defaultMode`: `"bypassPermissions"`

- `permissions.skipDangerousModePermissionPrompt`: `true`

- Do not write to `.claude/settings.json` — keep the shared file empty or absent.

- Append `.claude/settings.local.json` to `.gitignore`.

### Allow rules from Q6 (stack)

- 1 (Web/Node): `Bash(npm run lint)`, `Bash(npm run test:*)`, `Bash(npm run build)`, `Bash(npm run dev)`, `Bash(git status:*)`, `Bash(git diff:*)`, `Bash(git log:*)`.

- 2 (Python): `Bash(pytest:*)`, `Bash(python:*)`, `Bash(pip install:*)`, `Bash(git status:*)`, `Bash(git diff:*)`, `Bash(git log:*)`.

- 3 (Other / unsure): just `Bash(git status:*)`, `Bash(git diff:*)`, `Bash(git log:*)`.

### Sandbox block (from Q8 or auto-enabled in Path C)

If sandbox is on AND platform supports it, add:

```

"sandbox": {

  "enabled": true,

  "autoAllowBashIfSandboxed": true,

  "network": {

    "allowedDomains": [

      "github.com", "*.github.com",

      "*.npmjs.org", "registry.yarnpkg.com",

      "pypi.org", "*.pythonhosted.org",

      "files.pythonhosted.org", "crates.io"

    ]

  }

}

```

### Browser control (Q7 = 1)

- Create `.mcp.json` at repo root with the Playwright server (`npx @playwright/mcp@latest`).

- Set `"enableAllProjectMcpServers": true` in `.claude/settings.json`.

### Co-author (Q9 = 2)

- Add `"includeCoAuthoredBy": false`.
