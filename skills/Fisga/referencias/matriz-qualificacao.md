# Matriz de Qualificação — quão bom é esse lead?

Some os pontos abaixo comparando o prospect com o **Perfil de Cliente Ideal (ICP)** do config.
Ajuste os critérios à realidade do usuário — esta é a régua base.

## Pontuação

| Sinal | Pontos |
|------|:------:|
| Ramo bate com o nicho-alvo do ICP | +3 |
| Porte/faturamento na faixa que o usuário atende | +2 |
| Mostra um **gatilho de compra** ativo (contratando, expandindo, reclamação pública, post pedindo ajuda) | +3 |
| Dá pra alcançar quem decide diretamente (dono/sócio/gerente acessível) | +2 |
| Tem orçamento provável compatível com o ticket | +2 |
| Região / atende onde o usuário trabalha | +1 |
| Já demonstrou interesse no tema (curtiu, comentou, perguntou) | +2 |
| Concorrente direto do usuário | −10 |
| Claramente sem orçamento / fora da faixa | −4 |
| Sem nenhum gancho real encontrado | −3 |
| Já disse "não" antes / pediu pra não contatar | −20 |

## Faixas

- **🔥 Quente (7+):** prioridade. Aborda agora, gancho forte, follow-up firme.
- **🙂 Morno (3 a 6):** vale, mas capricha no gancho. Pesquisa mais antes de abordar.
- **❄️ Frio (abaixo de 3):** não gasta energia agora. Nutre (segue, comenta) ou descarta.

## Como usar numa lista
Pontue cada um, ordene do maior pro menor, e diga ao usuário **por onde começar** e **quem deixar
pra depois**. Energia de prospecção é escassa — vai onde converte.
