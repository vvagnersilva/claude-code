# Modelo de um bom POP (Procedimento Operacional Padrão)

Um POP é um passo a passo que **uma pessoa que nunca fez** consegue seguir
**sozinha**, sem o dono do lado, e chegar no mesmo resultado de sempre. Esse é o
único teste que importa: *uma pessoa nova consegue seguir isso e acertar?*

Esqueça linguagem chique. Um bom POP é curto, na ordem certa, e cada passo é uma
ação clara. Detalhe o suficiente pra não dar dúvida, enxuto o suficiente pra não
cansar.

## A estrutura (cada campo do POP)

| Campo | O que é | Exemplo |
|-------|---------|---------|
| **Nome** | Como o processo é chamado, curto e direto | "Fechar o caixa no fim do dia" |
| **Objetivo** | Para que serve / por que importa | "Garantir que o dinheiro do dia bate e nada fica solto" |
| **Quando** | O gatilho: quando esse processo roda | "Todo dia, na hora de fechar a loja" |
| **Responsável** | Quem faz (o papel, não a pessoa) | "Quem está no balcão no fechamento" |
| **Ferramentas** | O que usa | Maquininha, planilha do caixa |
| **Entradas** | O que precisa ter em mãos pra começar | Cupons do dia, saldo inicial da gaveta |
| **Passos** | O passo a passo, numerado, em ação | "1. Conte o dinheiro da gaveta…" |
| **Decisões** | "Se acontecer X, faça Y" | "Se faltar dinheiro, anote e avise o dono no mesmo dia" |
| **Qualidade** | Checklist de "como saber que ficou certo" | "[ ] Gaveta bate com o relatório" |
| **Não-fazer** | Erros comuns a evitar | "Nunca leve a diferença pra casa pra acertar depois" |
| **Saída** | O resultado final esperado | "Planilha do caixa do dia preenchida" |
| **Exemplo** | Um caso real recente | "Dia 12/06: gaveta R$ 840, relatório R$ 840, bateu" |
| **Nível** | Quão crítico (baixo/médio/alto) | alto |

## Como escrever os PASSOS (a parte que mais erra)

- **Comece com um verbo de ação:** "Confira…", "Envie…", "Anote…", "Pergunte…".
- **Um passo = uma ação.** Se um passo tem "e" no meio, provavelmente são dois.
- **Ordem importa.** Numere na sequência real.
- **Sem jargão interno solto.** Se usa um nome de sistema/atalho interno,
  explique na primeira vez.
- **Decisões saem do passo e viram bloco "Decisões".** O passo fica limpo; a
  ramificação ("se for cliente novo…") fica clara à parte.

❌ Ruim: "Faça o atendimento e veja se está tudo certo antes de liberar."
✅ Bom:
1. Cumprimente o cliente pelo nome (está na ficha).
2. Confirme o serviço que ele agendou.
3. Confira se o pagamento do agendamento já entrou.
4. Libere o atendimento.

## O CHECKLIST DE QUALIDADE é obrigatório

É o que separa um manual de verdade de um texto solto. Sempre feche o POP com
"como saber que ficou certo" — uma lista curta de itens conferíveis. É isso que
permite o dono **confiar** que, mesmo sem ele, o serviço sai no padrão.

Se o dono não souber dizer de cara, ajude: "no fim, o que você olha pra ter
certeza de que está tudo certo?"

## O bloco NÃO-FAZER evita o erro que sempre acontece

Todo dono tem aquele "ai, mas nunca faça isso". Capture. É o conhecimento que
normalmente só vem depois de alguém errar — colocar no POP economiza esse erro.

## Tamanho

- Processo simples: nome + objetivo + 3-6 passos + checklist. Pronto.
- Processo crítico (nível alto): vale caprichar nas decisões, no não-fazer e no
  exemplo — é onde um erro custa caro.
- Se passar de ~12 passos, provavelmente são **dois processos** — sugira quebrar.
