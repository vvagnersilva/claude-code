# Estruturas prontas — como montar o conteúdo de cada formato

Use no modo **Estruturar**. Pegue a expertise mapeada e devolva um esqueleto concreto,
módulo a módulo / capítulo a capítulo, na linguagem do dono. Nunca invente conteúdo que a
pessoa não domina — só organize o que ela sabe numa sequência que leva o aluno do "antes" ao
"depois" (a transformação).

Princípio que vale para TODOS: **uma promessa, um resultado.** O produto leva a pessoa de um
ponto A claro a um ponto B claro. Tudo que não serve a essa travessia, corta.

---

## Curso gravado — estrutura modular
```
Módulo 0 — Boas-vindas e o mapa (5-10 min)
   • o que você vai conseguir ao final (a transformação)
   • como aproveitar / o que ter em mãos
Módulo 1 — Fundamento (o "antes")
   • o conceito mínimo que destrava tudo
Módulos 2..N — Passo a passo da transformação
   • um módulo = uma etapa da jornada
   • cada aula: 1 ideia + 1 demonstração + 1 ação pro aluno fazer
Módulo Final — Colocar pra rodar / próximos passos
   • juntar tudo num resultado real
   • o convite pro próximo degrau (upsell natural)
```
Regra: aula curta (8-20 min), uma ideia por aula, sempre terminando com "agora faça X".

## Ebook / Guia — sumário
```
1. A dor (por que isso importa pra você, leitor)
2. O erro comum (o que quase todo mundo faz errado)
3. O caminho certo, passo a passo (o miolo — 3 a 7 passos)
4. Exemplo real / estudo de caso
5. Checklist de aplicação (1 página acionável)
6. Próximo passo (pra onde ir depois — convite pro produto maior)
```

## Template / Kit — o que entregar junto
```
• O arquivo/modelo em si (a peça central)
• Um guia rápido de uso (como preencher / adaptar) — 1 a 2 páginas
• 1 exemplo preenchido (mostra o "certo")
• (Opcional) vídeo curto de 3-5 min de walkthrough
```
Sem o guia de uso, o kit "não funciona" na cabeça do comprador. O guia É metade do valor.

## Desafio / Bootcamp — jornada com data
```
Dia 1  — Vitória rápida (a pessoa SAI com algo pronto já no 1º dia)
Dia 2..N-1 — uma etapa por dia, sempre com tarefa prática
Dia final — Juntar tudo + celebrar o resultado + convite pro próximo passo
```
A vitória rápida do dia 1 é o que segura a turma até o fim e gera prova social.

## Mentoria em grupo — desenho do programa
```
• Duração e ritmo (ex.: 8 encontros, 1 por semana, 60-90 min)
• Promessa do ciclo (onde o grupo chega ao final)
• Estrutura de cada encontro: tema + casos do grupo + tarefa
• O que está incluído: encontros + grupo de dúvidas + materiais
• O que NÃO está incluído (deixa explícito pra não virar suporte infinito)
```

## Comunidade / Assinatura — o que mantém viva
```
• Núcleo: o motivo de estar lá (conteúdo novo? troca? acesso a você?)
• Ritmo fixo: o que acontece TODA semana/mês (encontro, conteúdo, desafio)
• Trilha de entrada: o que o novo membro faz nos 7 primeiros dias (senão evade)
• Razões pra ficar: o que a pessoa perde se cancelar
```
Regra da assinatura: se você não consegue dizer o que entrega TODO mês, ainda não é uma
comunidade — é um curso. Reconsidere o formato.

---

## Checklist de qualidade da estrutura (antes de seguir)
- [ ] Dá pra resumir a promessa em uma frase ("ao final você consegue ___")?
- [ ] Cada módulo/capítulo é uma etapa da MESMA jornada (nada solto)?
- [ ] Tem uma vitória rápida cedo (pro aluno não desistir)?
- [ ] Todo conteúdo é algo que o dono REALMENTE domina (zero invenção)?
- [ ] Termina abrindo o próximo degrau (a escada de produtos)?
